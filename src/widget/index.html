OOPS: Creating Classes in Dojo

dojo/_base/declare provides features for class creation and subclassing. In this example we will demonstrate this by creating some custom widget classes and subclassing them

Firstly, an overview of the lifecycle of such a widget: 

* constructor (common to all prototypes, called when instantiated)
* postscript (common to all prorotypes built using declare
  * create
    * postMixInProperties
    * buildRendering
    * postCreate
  * startup

Let's go into the specifics of destruction now 

* destroyRecursive
  * destroyDescendants
  * destroy
    * uninitialize
    * destroyRendering

In order to verify all of this, we will manually define all of these functions in our widget, log out the name of the function, and call this.inherited(arguments) in order to ensure their original inherited invocation, which in this case delivers the following results
